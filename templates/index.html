Mission Statement

The purpose of Mirage Package Manager is to provide a simple and lightweight solution for dependency management for new developers. All you have to do to use Mirage is upload your project to Github, include a Mirage.toml file with build instructions for you project, register on this website and add your Github repo to your account.

Advantages of using Mirage:

Dev-oriented

Create and support packages for Debian-based and Arch-based systems at the same time.
Mirage offers developers a simple way to install, update, and support packages on many Linux distributions and other Unix-like systems. And it's much simpler to use tha

Great speed

Mirage has two global dependency: GoLang and Git. Compared to other package managers, Mirage is ultra fast and memory efficient.

Simple scaling

Mirage is completely decentralized: creating your own node is quite simple. All code is open source, and it follows FOSS principles (Mirage is licensed under WTFPL).

No data collection

Many programs today use telemetry for analyzing user data. Mirage, however, is built upon its community. We have an evaluation system that creates application ratings, public issues and much, much more.
